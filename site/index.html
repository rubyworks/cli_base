Executioner is a concise CLI framework for Ruby. A hierarchy of classes can define a complete multi-command command line tool using nothing more than Ruby's own method definitions. Commandline interfaces have never been so killer... `gem install execute`.

DESCRIPTION

Executioner was born out of a frustration with available command-line parsers. Prior art focused on special DSLs and few took good advantage of Ruby's inheritance system. Executioner address both of these issues by creating an OCM (Object Command Mapper).

SYNOPSIS

Executioner Base Class

Using Executioner is straight-forward. Simply subclass Executioner and add methods to handle command-line options and nested Executioner classes to handle sub-commands. A writer method (ending in ’=’) corresponds to an option. if a query method (ending in ’?’) of the same name exists then it is considered a boolean flag. And a bang method (ending in ’!’) indicates an "immediate" option.

For example, here is a simple commandline tool to run a Ruby script.

    require 'executioner'

    class Run < Executioner

      help "Require LIBRARY before executing your script"
      def require=(lib)
        require lib
      end
      alias :r= :require=

      help "Include PATH in $LOAD_PATH"
      def include=(path)
        $:.unshift path
      end
      alias :I= :include=

      help "Run in DEBUG mode"
      def debug=(bool)
        $DEBUG = bool
      end
      def debug?
        $DEBUG
      end

      help "Show this message"
      def help!
        puts self
        exit
      end
      alias :h! :help!

      def main(script)
        load(script)
      end
    end
